# 美食点评移动端 - 开发说明

## 📱 项目简介

这是一个基于 uni-app 开发的美食点评移动端应用，类似大众点评的功能。

## 🛠️ 技术栈

- **框架**: uni-app (Vue 3)
- **构建工具**: Vite
- **样式**: SCSS
- **UI风格**: 粘土风格 (Clay Design)
- **地图**: 高德地图API

## 🚀 开发环境

### 安装依赖
```bash
npm install
```

### 启动开发服务器 (H5)
```bash
npm run dev:h5
```

访问: http://localhost:5173

### 构建生产版本 (H5)
```bash
npm run build:h5
```

## 🔧 配置说明

### API配置
在 `src/api/request.js` 中配置后端API地址：

```javascript
const BASE_URL = {
  development: 'http://localhost:8080/api',
  production: 'https://api.business-reviews.com/api'
}
```

### 代理配置
在 `vite.config.js` 中配置开发代理：

```javascript
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8080',
      changeOrigin: true
    }
  }
}
```

## 🎯 开发注意事项

### 1. 网络请求
- 所有API请求都通过 `src/api/request.js` 统一封装
- 自动处理token认证
- 自动处理401未登录跳转

### 2. 样式规范
- 使用rpx作为单位（响应式像素）
- 全局样式在 `src/static/styles/common.scss`
- 采用粘土风格设计（Clay Design）

### 3. 图标资源
- 所有图标存放在 `src/static/icons/`
- 使用PNG格式
- 命名规范：功能名.png, 功能名-active.png

### 4. 地图功能
- 使用高德地图API
- Key: `1521141ae4ee08e1a0e37b59d2fd2438`
- 支持定位、逆地理编码等功能

## 🐛 常见问题

### Q: 网络请求失败
A: 检查后端服务是否启动，确认API地址配置正确

### Q: 图片无法显示
A: 检查图片路径是否正确，确保图片文件存在

### Q: 定位功能不工作
A: H5环境需要HTTPS或localhost才能使用定位功能
