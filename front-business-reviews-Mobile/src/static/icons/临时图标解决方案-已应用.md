# 临时图标解决方案 - 已应用

## 问题说明
由于创建的 PNG 图标文件是文本占位符而不是真正的图片文件，导致底部导航栏图标无法显示。

## 已应用的临时解决方案

### 1. 底部导航栏（tabBar）
**修改位置**：`src/pages.json`
**解决方案**：使用 Unicode Emoji 字符替代图标

```json
"list": [
  { "pagePath": "pages/index/index", "text": "🏠 首页" },
  { "pagePath": "pages/map/map", "text": "🎫 优惠" },
  { "pagePath": "pages/publish/publish", "text": "➕ 发布" },
  { "pagePath": "pages/message/message", "text": "💬 消息" },
  { "pagePath": "pages/profile/profile", "text": "👤 我的" }
]
```

### 2. 首页搜索和箭头图标
**修改位置**：`src/pages/index/index.vue`
**解决方案**：使用 Unicode 字符替代图片

- 下拉箭头：`▼`
- 搜索图标：`🔍`

## 当前状态
✅ 底部导航栏正常显示（使用 Emoji 图标）
✅ 首页搜索框和城市选择正常显示
✅ 分类图标正常显示（使用现有 PNG 文件）

## 恢复到图片图标的方法

### 方法1：替换 PNG 文件
1. 准备真正的 PNG 图标文件（48x48 或 64x64 像素）
2. 替换 `src/static/icons/` 目录中的占位符文件
3. 恢复 `pages.json` 中的 iconPath 配置

### 方法2：使用在线工具转换
1. 将现有的 SVG 文件转换为 PNG 格式
2. 使用工具：https://convertio.co/svg-png/
3. 下载并替换到 icons 目录

### 方法3：使用 iconfont 字体图标
1. 从 iconfont.cn 下载图标字体
2. 在项目中引入字体文件
3. 修改代码使用字体图标
