# 店铺信息页面UI优化说明

## 更新日期
2025-12-22

## 更新内容

### 1. 页面结构优化
将原本分离的"基本信息"和"联系方式与地址"两个卡片合并为一个统一的"基本信息"卡片，提升页面整体性和用户体验。

### 2. 布局调整

#### 合并前结构：
```
┌─ 基本信息 ─────────────────┐
│ - 店铺封面                  │
│ - 店铺名称、经营类目        │
│ - 店铺简介、商家相册        │
│ - 人均消费、店铺状态        │
└────────────────────────────┘

┌─ 联系方式与地址 ───────────┐
│ - 联系电话、营业时间        │
│ - 店铺位置、经纬度          │
└────────────────────────────┘
```

#### 合并后结构：
```
┌─ 基本信息 ─────────────────┐
│ - 店铺封面                  │
│ - 店铺名称、经营类目        │
│ - 店铺简介、商家相册        │
│ - 人均消费、店铺状态        │
│                             │
│ ─── 联系方式与地址 ───      │
│                             │
│ - 联系电话、营业时间        │
│ - 店铺位置                  │
└────────────────────────────┘
```

### 3. 视觉优化

#### 分隔线设计
- 添加了优雅的渐变分隔线，区分基本信息和联系信息
- 分隔线中间显示"联系方式与地址"文字标签
- 使用渐变效果，两端透明，中间实线

#### 经纬度处理
- **隐藏显示**：经纬度输入框改为 `type="hidden"`，用户界面不再显示
- **保留功能**：
  - 数据仍通过 `v-model` 绑定到 `shopForm.longitude` 和 `shopForm.latitude`
  - 定位按钮自动获取并保存经纬度
  - 地址解析自动填充经纬度
  - 地图选点自动保存经纬度
  - 后端提交时经纬度数据正常传递

#### 定位按钮优化
- 添加了图标和文字，更加直观
- 优化了按钮样式和间距
- 保持了原有的加载状态显示

## 优势

1. **视觉统一**：所有基本信息集中在一个卡片中，视觉更加统一
2. **信息层次**：通过分隔线清晰区分不同类型的信息
3. **简洁美观**：减少了卡片数量，页面更加简洁
4. **用户体验**：信息集中，用户无需在多个卡片间切换
5. **功能完整**：所有功能保持不变，经纬度隐藏但功能正常
