# 商家设置功能实现总结

## 1. 概览
响应用户需求，我们升级了商家运营中心的"商家信息"设置页面，使其与商家入驻时的信息保持一致，并支持"先修改后保存"的防误触模式。
特别调整：**移除了商家Logo修改**，因为Logo在业务逻辑中作为门店封面图，应在门店管理中维护。

**支持修改的字段**:
1.  **商家头像** (`avatar`)
2.  **商家名称** (`merchantName`)
3.  **商家姓名** (`merchantOwnerName`)
4.  **联系电话** (`phone`)
5.  **联系邮箱** (`contactEmail`)
6.  **营业执照号** (`licenseNo`)
7.  **营业执照** (`licenseImage`)

## 2. 前端实现 (`src/views/settings/index.vue`)

### 2.1 交互模式更新
- **编辑模式**: 默认处于查看模式（所有字段只读，外观与普通文本一致）。
- **修改按钮**: 在标题栏右侧新增"修改"按钮。点击后进入编辑模式，字段变为可输入，并在下方显示"保存修改"按钮。
- **取消修改**: 编辑模式下显示"取消"按钮，点击可还原数据并退出编辑模式。

### 2.2 字段与样式
- **移除Logo**: 删除了Logo上传组件及相关逻辑。
- **正常样式**: 只读状态下，输入框保持正常背景和文字颜色（使用 `readonly` 而非 `disabled`），避免灰色不可点击的视觉感受。
- **上传控制**: 非编辑模式下，点击上传区域无效。

## 3. 后端实现

### 3.1 DTO (`MerchantUpdateDTO`)
- 移除了 `logo` 字段。

### 3.2 Service (`MerchantAuthServiceImpl`)
- 移除了 `logo` 字段的更新逻辑。

## 4. 验证与测试
1. 进入"系统设置" -> "商家信息"。
2. 页面默认展示信息，无法直接编辑，无灰色禁用样式。
3. 点击标题栏右侧"修改"按钮。
4. 字段变为可编辑，上传组件激活。
5. 修改信息后点击"保存修改"，保存成功并自动返回查看模式。
6. Logo相关信息不再显示。
