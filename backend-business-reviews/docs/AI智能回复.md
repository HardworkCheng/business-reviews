你是一位精通 Java Spring Boot 和 LangChain4j 的后端开发专家。请帮我实现商家端的一个核心功能：“差评智能回复助手”。

### 1. 业务场景
当商家收到用户的差评时，往往情绪激动或不知道如何妥善处理。该功能允许商家输入“简单的补偿策略”（例如：送一张8折券），AI 自动结合用户的差评内容，生成一段**高情商、礼貌、具有安抚性质**的回复文案。

### 2. 技术栈
- **框架**: Spring Boot 3.x(基于当前项目)
- **AI 框架**: LangChain4j (基于当前项目)
    - 依赖: `langchain4j-spring-boot-starter-ai-services`
    - 依赖: `langchain4j-open-ai-spring-boot-starter`
- **语言**: Java 17+

### 3. 代码生成要求
请按照以下步骤生成完整的代码：

**Step 1: 定义 AI 服务接口 (ReplyAgent)**
创建一个接口 `ReviewReplyAgent`，使用 `@AiService` 注解。
- **System Message (人设)**: 
    - 你是校园商圈的金牌客服经理，擅长危机公关和情绪安抚。
    - 你的语气必须诚恳、温暖、专业，绝对不能推卸责任。
    - 回复结构要求：1. 真诚致歉 -> 2. 共情用户的不满（复述问题） -> 3. 提出具体的改进措施 -> 4. 给出补偿方案（如果有） -> 5. 再次致歉并期待回归。
- **User Message (输入)**: 
    - 接收两个参数：`customerReview` (用户的差评内容) 和 `compensationStrategy` (商家的补偿策略，如"送一张5元券"，如果为空则不提补偿)。
- **Output**: 返回生成的回复字符串 (`String`)。

**Step 2: 创建 Controller/Service**
创建一个 `MerchantReplyController` 和对应的 Service。
- 提供一个 POST 接口 `/merchant/reply/generate`。
- **Request Body**: 包含 `reviewText` (用户评论) 和 `strategy` (商家想给的优惠，可选)。
- **逻辑**: 调用 `ReviewReplyAgent` 生成回复并返回给前端。

**Step 3: 异常处理与兜底**
- 简单的处理逻辑：如果 `strategy` 字段为空，AI 的回复中不应该包含关于优惠券的虚假承诺。

### 4. 期望的回复示例 (供参考)
- 输入: 
    - 差评: "空调太热了，根本吃不下去饭！"
    - 策略: "送两张冰粉券"
- 期望输出: 
    - "亲爱的同学，真的非常抱歉给您带来了这么糟糕的用餐体验！😭 看到您反馈空调太热影响了吃饭的心情，我们心里也很过意不去。我们已经立刻联系物业检查空调设备了。为了表达我们的歉意，特意为您准备了两张冰粉券，希望能为您消消暑。真的希望您能再给我们一次机会！"

请生成包含 import 语句的完整 Java 代码。



### 💡 核心设计思路解读 (为什么这样写 Prompt？)

1. **人设分离 (`@SystemMessage`)**: 我在提示词中强调了“金牌客服经理”的人设，并规定了**5步回复法**（致歉-共情-改进-补偿-结尾）。这能保证无论用户骂得多难听，AI 输出的永远是标准的“公关文”。
2. **变量控制 (`compensationStrategy`)**: 这是这个功能的灵魂。
   - **如果商家只说“送券”** -> AI 会把这句话扩写成“为您准备了优惠券作为小小弥补”。
   - **如果商家什么都不给** -> AI 依然会道歉，但不会胡乱许诺给钱，避免造成商家损失。
3. **情感共鸣**: 我在 Prompt 示例中加了 Emoji (😭)，这对于校园类应用非常重要，能拉近与大学生的距离，显得不那么像机器人。
